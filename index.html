How I Think as a Salesforce Developer
🎯
Requirements before code
Map business process first — objects, data volume, user roles. Design for 10,000 records from day one, not 10.
⚖️
Declarative-first principle
If a Flow solves it cleanly — use it. Apex is a deliberate choice for bulk operations or complex logic, not the default.
📏
Governor limits by design
Plan around 150 SOQL / 10K DML / 100 callout limits before writing line one. Async Apex is first-class, not a hotfix.
🔒
Security model is non-negotiable
CRUD, FLS, and sharing rules by design. Deliberate with/without sharing decisions in every Apex class.
🔍
Debug before guess
Developer Console logs, structured System.debug, test class isolation — understand root cause before adding workarounds.
🔄
Maintenance is a feature
Thin triggers, named constants, test methods describing behavior not coverage. Code is read 10× more than it's written.
